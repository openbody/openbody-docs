OpenBody uses **three licenses, one per artifact kind**, so the interop core stays maximally
unencumbered while the steward keeps a royalty-free patent posture. This summarizes
`LICENSING.md`, which is authoritative.

| Artifact | License | Why |
|---|---|---|
| **Specification documents** (`SPEC.md` and the prose docs) | **Open Web Foundation Agreement 1.0 (OWFa 1.0)** | Reciprocal royalty-free patent licensing with defensive termination — the standards-body norm for an open spec. |
| **Reference code** (SDKs, mappers, conformance tooling) | **Apache-2.0** | Permissive, with an explicit patent grant. |
| **Registry data** (the exercise & measurement-type registries) | **CC0** | Public domain; the interop registries must be completely unencumbered — no copyleft contamination. |

<Aside type="note" title="Why three, not one">
A standard has to be **fully open, not open-core**, to be adopted — the entire model and its
mapping rules are public. Splitting by artifact lets the *spec* carry a reciprocal patent
posture (OWFa), the *code* stay permissive (Apache-2.0), and the *registry* be pure public
domain (CC0) so it can be embedded anywhere without contaminating downstream licenses.
</Aside>

## Official license texts

- **OWFa 1.0** — [openwebfoundation.org](https://www.openwebfoundation.org/the-agreements/the-owf-1-0-agreements-granted-claims/owfa-1-0)
- **Apache-2.0** — [apache.org/licenses/LICENSE-2.0](https://www.apache.org/licenses/LICENSE-2.0)
- **CC0 1.0** — [creativecommons.org/publicdomain/zero/1.0](https://creativecommons.org/publicdomain/zero/1.0/legalcode)

## Vendored texts

Each repository vendors the verbatim text of its license:

- **Reference code** (`openbody-ts/LICENSE`) — Apache-2.0, full text.
- **Registry data** (`openbody-registry/LICENSE`) — CC0 1.0 legal code, full text. Both
  registries (exercise and measurement-type) live in this repository.
- **Specification docs** (`openbody/LICENSE`) — the Open Web Foundation **Final
  Specification Agreement (OWFa 1.0), Patent and Copyright Grants** variant (not the
  Patent-Only or CLA variants), with the Specification identified as `OpenBody 0.8.0` and an
  individual grant executed.

<Aside type="caution" title="This site vs. the licenses">
These pages describe and link the licenses for convenience; the **vendored `LICENSE` files
in each repository govern**. The earlier CC-BY-4.0 docs license was superseded by OWFa 1.0
as of 2026-06-24; the OWFa grant is re-executed for each released spec version — currently
`OpenBody 0.8.0` (2026-07-03), superseding `OpenBody 0.6.0` (2026-07-01) and the initial
`OpenBody 0.3.1`.
</Aside>
